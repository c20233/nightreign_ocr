# 実装完了の確認 (Walkthrough)

ご提示いただいた画面画像（右下の赤枠部分：遺物の詳細ステータス）に基づき、前処理ロジックとクレンジング処理の実装を完了しました。

## 実装内容のサマリー

### 1. 関心領域（ROI）の再調整
画像から「画面右下」に位置することが確認できたため、1920x1080解像度を想定し、おおよその座標に調整しました（環境や実際の録画サイズに合わせて適宜微調整が必要です）。
- `y_start, y_end = 650, 1000`
- `x_start, x_end = 900, 1850`

### 2. 前処理（`preprocess_image`）の実装
画像内の文字色が主に「白色〜明るいグレー」および「薄い青色（デメリット効果）」であることを考慮し、以下の処理を組み込みました。
- **HSVカラーマスク:** 
  - **白・明るいグレー:** 明度が非常に高く、彩度が低い部分
  - **薄い青色:** 色相（Hue）がシアン〜青（90~130付近）で明度が高い部分
  上記2つのマスクを合成（`cv2.bitwise_or`）し、背景の青暗いグラデーションなどを黒く塗りつぶします。
- **適応的2値化 (`cv2.adaptiveThreshold`):** HSVで抽出した画像をグレースケール化し、局所的な明暗差に強い適応的2値化で文字のエッジを際立たせます。
- **モルフォロジー変換:** `cv2.morphologyEx` を用いて、点ノイズの除去と細い文字の膨張（見切れの補修）を行います。

### 3. クレンジング・パース処理（`clean_ocr_text`, `parse_extracted_text`）
誤認識に対処するため、以下のロジックを実装しました。
- **NFKC正規化:** `unicodedata` を使い、全角数字やアルファベットを半角に統一。
- **典型的な誤認識の置換:** `l`, `I`, `|` などを `1` に、`O`, `o` を `0` に置換する辞書マップを導入。
- **柔軟なパースと正規表現:** 
  - 画像内の「アーツゲージ蓄積増加＋３」のように『文字列＋数値』となるパターンを確実に取り出すため、正規表現 `r'(.+?)[+＋]([-+]?\d+)'` を使用し、効果名と数値（JSON上の `effects` 配列）に正しく分離します。
  - 「技量と信仰が低下」などのように**数値を持たないテキスト**（デメリット効果や特殊効果）についても、テキストそのままを `effects` としてパースし、取りこぼしを防ぐように拡張しました。
