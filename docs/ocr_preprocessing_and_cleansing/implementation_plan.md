# 画像前処理の強化とOCRデータのクレンジング実装計画

このドキュメントでは、ゲーム特有の派手なエフェクトやノイズに対応するための画像前処理ロジックの強化案と、OCR誤認識を補正するテキストクレンジングのベストプラクティスについて提案・計画します。

## 1. 解決すべき課題とアプローチ
- **前処理の課題**: 背景エフェクトとテキストの分離が難しい。固定閾値の2値化(`cv2.threshold`)では、明暗のグラデーションや半透明ウインドウに対応しきれない。
- **テキスト解析の課題**: OCR特有の誤認識（1とl、0とOの混同）や、全角/半角の揺れが生じ、データが扱いづらくなる。

## 2. 提案する変更内容

### A. 前処理ロジックの強化（`cv2`の活用）
既存のグレースケール＋単純2値化から、以下の複合パイプラインへとアップデートします。

1. **HSVカラーマスクによる色抽出**
   - 対象となる文字色（例：白、金、ステータスの緑など）のみを抽出するマスクを作成します。
   - `cv2.cvtColor(img, cv2.COLOR_BGR2HSV)` と `cv2.inRange()` を組み合わせることで、指定色以外（背景エフェクトなど）を黒く塗りつぶし、ノイズを劇的に減らします。
2. **適応的2値化（Adaptive Thresholding）**
   - 画面内に明暗差がある場合に対応するため、局所的なしきい値で2値化を行う `cv2.adaptiveThreshold()` を用います。これによりグラデーション背景でも文字が鮮明に浮かび上がります。
3. **モルフォロジー変換（ノイズ除去・文字の結合）**
   - 細いフォントのカスレを補修するための膨張処理(`cv2.dilate`)や、微小な点ノイズを消す処理(`cv2.morphologyEx`)を挟むことで、PaddleOCRの認識率を底上げします。

### B. テキストクレンジングのベストプラクティス（パース処理）
認識後のテキストに対して、以下の段階的なクレンジングパイプラインを実装します。

1. **文字種の正規化 (NFKC)**
   - `unicodedata.normalize('NFKC', text)` を使用し、全角英数字（例：「１」、「Ａ」）を半角に統一します。
2. **ルールベースの誤認識置換（マッピング）**
   - 数値が入るべき箇所でよくある誤認識（`l, I, | -> 1`, `O, o -> 0`, `S -> 5`, `Z -> 2`, `, -> .` など）を `str.translate` や `replace` で強制補正します。
3. **キー（項目名）のファジーマッチング**
   - 「攻撃力」が「攻繋力」のように誤認識された場合に対処するため、あらかじめゲーム内の正しい項目名リスト（マスターデータ）を定義しておき、`difflib.get_close_matches` などを利用して最も近い正しい名前に自動補正します。
4. **正規表現（Regex）による確実な値の抽出**
   - コロン `:` 区切りのパースだけでなく、`re.findall(r'[-+]?\d*\.?\d+', val_str)` などの正規表現を用いて、テキスト内から数値部分のみを安全に抽出します。
