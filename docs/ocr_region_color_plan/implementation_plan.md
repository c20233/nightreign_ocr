# 実装計画: OCR読み取り領域の4分割と効果のメリット/デメリット自動判定

ユーザー様より提供された新しい `sample.png` の「緑の枠」に基づき、読み取り領域を4つに分割して個別にOCRをかけ、文字色からメリット/デメリットを判定する仕組みを実装します。

## 提案される変更内容

### 1. 読み取り領域（ROI）の4分割
`sample.png` の緑枠を解析した結果、以下の4つの領域座標が特定できました。これらの座標を `main.py` にハードコードして使用します。

- **タイトル (Title):** `y=768~805`, `x=1067~1806`
- **効果1 (Effect 1):** `y=810~862`, `x=1067~1806`
- **効果2 (Effect 2):** `y=868~922`, `x=1067~1806`
- **効果3 (Effect 3):** `y=929~981`, `x=1067~1806`

### 2. 超高速化（画面変化検知）の維持
現在の「画面の変化を自動検知してOCRをスキップする機能」はそのまま維持します。タイトルから効果3までの全体を含む大きな領域で差分を監視し、変化があった時のみ、上記の4つの枠に対して個別にOCRを実行します。

### 3. メリット（白）/ デメリット（水色）の自動判定
効果1〜3のそれぞれの枠に対して、OpenCV を使って「文字の色（最も明るいピクセルの色合い）」を解析する関数 `classify_text_color` を実装します。
- **彩度(Saturation)が低い場合** → 白色と判定し、`"merit"`
- **彩度が高く、色相(Hue)が水色〜青（80〜130付近）の場合** → 水色と判定し、`"demerit"`

### 4. JSON出力フォーマットの再構築
ご要望に合わせて、`game_stats.json` に出力されるデータの構造を以下のように変更します。

```json
{
    "frame": 60,
    "stats": {
        "title": "読み取ったタイトルテキスト",
        "effect1": {
            "type": "merit", 
            "text": "読み取った効果1のテキスト"
        },
        "effect2": {
            "type": "demerit", 
            "text": "読み取った効果2のテキスト"
        },
        "effect3": {
            "type": "merit", 
            "text": "読み取った効果3のテキスト"
        }
    }
}
```
※枠内にテキストが存在しない場合（効果が1つしかない場合など）は、該当の effect項目 は出力しない、もしくは `null` になります。

## 確認事項
1. 上記のJSONフォーマットはご希望の形に沿っていますでしょうか？
2. デメリットの水色判定のしきい値は一般的な水色（Cyan〜Blue）を想定していますが、もし実際のゲーム画面で判定漏れが発生した場合は、後から微調整をかけさせていただきます。
