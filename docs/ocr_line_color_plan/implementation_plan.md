# 実装計画: 行単位の文字色判定とメリット/デメリットの分離

ご提案いただいたロジックは非常に理にかなっています！
PaddleOCRはそもそも1行ごとに別々のテキストと座標（矩形）を返してくれます。そのため、「枠全体」の色を見るのではなく、「読み取った1行ごとの画像」を切り抜いて色を判定することで、完全に正確な色判定が可能になります。

## 提案される変更内容

### 1. 行ごとの色判定の導入
OCRがテキストを読み取った際、そのテキストの座標（Bounding Box）を使って、画像から「その1行だけの画像」を切り抜きます。
切り抜いた1行単位の画像に対して `classify_text_color` を実行し、それぞれの行が「merit（白）」か「demerit（水色）」かを判定します。

### 2. 結合と分離のロジック（ユーザー様のご提案を採用）
各枠（effect1, effect2, effect3）において、複数の行が検出された場合：
- **すべて同じ色の場合**: 1つの効果としてテキストを結合し、その色を設定する。
- **色が異なる場合（1行目がmerit、2行目がdemerit等）**: テキストを結合せず、それぞれ別の効果として分離して出力する。

### 3. JSON出力フォーマットの変更案
1つの枠からメリットとデメリットの両方が出現する可能性があるため、`effect1` などのキーに対して、リスト（配列）形式で出力する形に変更することを提案します。

**変更後のJSONイメージ：**
```json
{
    "frame": 253,
    "stats": {
        "title": "壮大な滴る雷景",
        "effect1": [
            {
                "type": "merit",
                "text": "敵を倒した時、アーツゲージ増加+1"
            },
            {
                "type": "demerit",
                "text": "取得ルーン減少"
            }
        ],
        "effect2": [
            {
                "type": "merit",
                "text": "周囲で発狂状態の発生時、攻撃力上昇+1"
            },
            {
                "type": "demerit",
                "text": "筋力と知力が低下"
            }
        ],
        "effect3": [
            {
                "type": "merit",
                "text": "潜在する力から、フレイルを見つけやすくなる"
            }
        ]
    }
}
```
